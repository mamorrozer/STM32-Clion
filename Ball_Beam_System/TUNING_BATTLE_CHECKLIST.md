# Ball-Beam PID 调参作战清单

> 目标：按固定流程快速收敛，避免“凭感觉反复试错”。

## 1. 调参前准备（必须先做）

- [ ] 机械中位确认：舵机 50° 时横梁接近水平、球不会自动滚落。
- [ ] 传感器静态检查：球不动时测距抖动在可接受范围（不要频繁跳大值）。
- [ ] 安全边界确认：舵机最小/最大角度不会顶死机构。
- [ ] 串口日志确认：能看到 `SETPOINT/MEASUREMENT/ERROR/CONTROL_OUTPUT/ANGLE/MODE/STATE`。

## 2. 调参作战清单（按顺序，不要跳）

- [ ] 第一步只调 P：`Ki=0, Kd=0`，从小到大加 `Kp`，找到“刚开始持续振荡”的点，再回退到其 50%~70%。
- [ ] 第二步加 D：固定 `Kp`，小步增加 `Kd`，目标是减小超调和来回摆动；出现高频抖动就回退一点 `Kd`。
- [ ] 第三步加 I：在 `Kp/Kd` 稳定后，小步增加 `Ki`，只为消除稳态误差；若出现慢速大幅摆动，减小 `Ki`。
- [ ] 调 `PID_ENABLE_DELAY_MS`：若开机初期不稳，延长延时，让系统先 PD 稳住再进 PID。
- [ ] 调 `PID_INTEGRAL_ENTER_BAND_* / PID_INTEGRAL_EXIT_BAND_*`：给积分加进入/退出滞回，避免在边界附近反复开关；一般先让 `EXIT > ENTER`，再按振荡情况微调。
- [ ] 调 `PID_INTEGRAL_LIMIT`：防积分堆太多；若恢复慢且有“拖尾”，先减小它。
- [ ] 调 `SERVO_SLEW_MAX_STEP_DEG`：抖动大就减小，响应太慢就略增。
- [ ] 调 `SERVO_MIN_STEP_DEG`：若中心附近细小修正无效，按舵机实际分辨率设置。
- [ ] 调 `SENSOR_MAX_STEP_MM`：误判无效数据多就放宽一点，噪声穿透多就收紧一点。

## 3. 每轮调参固定测试（每改一次参数都做）

- [ ] 中心保持测试（`setpoint=0`）。
- [ ] 定点保持测试（例如 `+60mm / -60mm`）。
- [ ] 扰动恢复测试（手动扰动后回中心）。
- [ ] 记录三项：稳定时间、稳态误差、是否持续振荡。

## 4. 建议验收线（调完 PID 后先过这个）

- [ ] 稳定时间 ≤ 8s。
- [ ] 稳态误差在 ±10mm 内（持续观察一段时间）。
- [ ] 无明显持续振荡，且不会频繁进入 `FAULT`。

## 5. 调完 PID 后下一步执行计划

- [ ] 固化参数：把最终参数写入代码常量，并记录测试日期与工况。
- [ ] 做长稳测试：连续运行（建议 30~60 分钟）观察温漂、抖动、故障率。
- [ ] 做边界测试：不同初始位置、轻微外力、不同目标点，确认都可收敛。
- [ ] 优化日志与指标：保留关键遥测，统一达标判断标准。
- [ ] 再做结构优化：验收通过后再考虑目标斜坡、前馈、状态机细化等策略。
- [ ] 最终交付：整理“最终参数表 + 测试结果 + 已知限制 + 下一版改进点”。
